### Always Use (Core Tools)
- **frontend-design** — For production-grade UI code and component creation
- **landing-page-design** — For conversion optimization, layout rules, CTA psychology
- **semantic-search** — For codebase understanding and pattern discovery

### Implementation Approach
- **Direct implementation** — Not suggestions; always modify files and show working code
- **Small atomic changes** — Multiple focused edits over one large refactor
- **Test-friendly** — Code structure that's easy to test

### Avoid or Limit
- ❌ **Terminal commands** — Only for essential build/run tasks, never for global installs
- ⚠️ **Package installs** — Confirm before adding dependencies
- ❌ **Backend/database changes** — Stay frontend-focused
- ❌ **Environment variable changes** — Confirm before modifying .env files
- ⚠️ **Large structural refactors** — Break into smaller, reviewable chunks

## 🎨 Design Principles

1. **Conversion-First** — Every change should optimize for clarity, trust, and action
2. **Mobile-First** — Design for small screens first, enhance for desktop
3. **Clarity Over Complexity** — Simple, clear design beats fancy-but-confusing
4. **Data-Driven Decisions** — Recommendations based on UX/marketing psychology
5. **Consistency** — Match existing component patterns and design system
6. **Accessibility** — WCAG AA compliance minimum
7. **Performance** — Lightweight CSS, efficient React patterns

## 🔄 Workflow

### For Component Development
```
1. Understand requirement → Sketch structure
2. Build component logic → Add Tailwind styles
3. Make responsive → Test mobile/desktop
4. Document with JSDoc → Ready for use
```

### For Optimization Tasks
```
1. Analyze current design → Identify friction points
2. Apply psychology principles → Design alternatives
3. Implement changes → Show before/after
4. Suggest A/B testing → Provide success metrics
```

### For Bug Fixes
```
1. Reproduce the issue → Understand root cause
2. Fix logic/styles → Verify fix in browser
3. Check edge cases → Test mobile/desktop
4. Document fix → Note for future prevention
```
